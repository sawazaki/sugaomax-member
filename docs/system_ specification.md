# 菅生マックス チーム管理システム 仕様書

Version 3.3 / 更新日: 2026-04-17

---

## 1. システム概要

菅生マックス（ミニバスケットボールチーム）の部員情報を一元管理し、試合ごとの公式メンバー表をブラウザ上で作成・印刷できるWebアプリケーションです。

### 目的

- Excelや手書きによるメンバー表作成作業を廃止し、ブラウザから誰でも簡単に作成できるようにする
- 部員情報（氏名・学年・背番号・性別・所属校・身長・ビブス番号等）を一元管理する
- 複数のコーチ・保護者が同じデータにアクセスできる共有管理を実現する
- 試合記録を蓄積し、過去のメンバー表をいつでも参照できる
- 公式バスケットボールスコアシート様式で印刷できる
- 練習・試合の当番をグループ分けし、ドラッグ&ドロップで管理できる
- webcalカレンダー（BAND等）と連携したA4横活動予定シートをJPGで出力できる
- 保護者がQRコード・パスワードで入部届けをWeb上で提出できる

### 技術スタック

| 項目           | 内容                                           |
| -------------- | ---------------------------------------------- |
| 言語           | PHP 8.2                                        |
| データベース   | SQLite 3（ファイルベース、別途DBサーバー不要） |
| フロントエンド | HTML5 / CSS3 / Vanilla JavaScript              |
| 動作環境       | Apache on Linux (Docker / Xserver)             |
| 認証方式       | セッションベース ロール別パスワード認証         |
| 対応ブラウザ   | Chrome 推奨（印刷機能はChrome使用を前提）      |

> **DATA_DIR の自動検出**: `includes/db.php` は起動時に `includes/` の2つ上のディレクトリ名が `public_html` かどうかで動作環境を自動判別します。
> - **Xserver**: `…/sugaomax.com/public_html/` の場合 → `data/` は `…/sugaomax.com/data/`（`public_html` と同列）
> - **Docker / 開発環境**: `/var/www/html/` の場合 → `data/` は `/var/www/html/data/`
>
> これにより FTP で `public_html` を上書きしても `data/` は影響を受けません。

## 4. データベース仕様

| `grade`                    | INTEGER | ○    | 学年（1〜6）                                   |

## 5. 認証・権限仕様

### 5.1 ロール一覧

| ロール  | 定数キー                | 説明                                                             |
| ------- | ----------------------- | ---------------------------------------------------------------- |
| admin   | `ADMIN_PASSWORD_HASH`   | 全操作可能。設定画面（パスワード変更）へのアクセス権を持つ       |
| editor  | `APP_PASSWORD_HASH`     | 部員・試合・当番・入部届けの編集が可能。設定画面は不可           |
| viewer  | `VIEWER_PASSWORD_HASH`  | データの閲覧のみ可能。編集・追加・削除不可（省略可）             |

- ログイン成功後、`$_SESSION['logged_in'] = true` および `$_SESSION['role']` にロール名をセット
- 全ページの先頭で `require_login()` を呼び出し、未認証の場合は `login.php` へリダイレクト
- 編集操作が必要なページは `require_editor()`、管理者専用ページは `require_admin()` を使用

### 5.2 パスワードの保存方式

| 項目                 | 内容                                             |
| -------------------- | ------------------------------------------------ |
| 保存先               | `data/config.php`（Git管理外）                   |
| ハッシュアルゴリズム | bcrypt（`PASSWORD_BCRYPT`、cost=12）             |
| 検証関数             | `password_verify()`                              |
| 最小文字数           | 8文字                                            |
| 定義定数             | `ADMIN_PASSWORD_HASH` / `APP_PASSWORD_HASH` / `VIEWER_PASSWORD_HASH` / `ENROLLMENT_PASSWORD_HASH` / `ENROLLMENT_TOKEN` / `ENROLLMENT_ACTIVE` |

### 5.3 初回セットアップ

`data/config.php` が存在しない場合、全ページへのアクセスが自動的に `setup.php` へリダイレクトされます。`setup.php` でパスワードを入力すると `data/config.php` が生成されます。

### 5.4 パスワードの変更

ログイン後、`settings.php`（管理者のみ）から各ロールのパスワードを変更できます。

### 5.5 CSRFトークン

フォーム送信・AJAX処理には全てCSRFトークンによる検証（`verify_csrf()`）を適用します。

## 6. 部員管理機能

### 6.1 部員一覧（members.php）

- **性別タブ**: ALL / 男子 / 女子（各タブに件数表示）
- **所属校タブ**: ALL / 犬蔵小学校 / 菅生小学校 / 稗原小学校 / その他（各タブに件数表示）
- 2つのタブは**組み合わせフィルター**対応（例: 男子 + 犬蔵小学校）
- **ソート機能**: 学年・背番号・氏名・性別・所属校・リバビブ・青ビブ（昇順/降順切替）
- デフォルト並び順: 学年降順（6年から）
- 氏名クリックで部員編集ページへ遷移
- 表示列: 学年・背番号・氏名（姓＋名）・性別・所属校・リバーシブルビブス・青ビブス

### 6.2 部員追加（member_add.php）

入力フィールド:

| フィールド             | 必須 | 備考                   |
| ---------------------- | ---- | ---------------------- |
| 姓                     | ○    |                        |
| 名                     |      |                        |
| ローマ字表記           |      | 名前のみ（例: TARO）   |
| 学年                   | ○    | 1〜6年                 |
| 性別                   |      | 男子 / 女子            |
| 背番号                 |      | 0〜99                  |
| 所属小学校             |      |                        |
| 身長（cm）             |      | 80〜220                |
| リバーシブルビブス番号 |      | 0〜99                  |
| 青ビブス番号           |      | 0〜99                  |
| 練習当番               |      | A〜J                   |
| 試合当番               |      | 1〜4                   |
| 入部日                 |      | カレンダー入力（YYYY-MM-DD） |
| 兄弟フラグ             |      | 兄弟在籍の場合チェック |

### 6.3 部員編集（member_edit.php）

- 部員追加と同じ基本フィールドを編集可能（入部日フィールドを含む）
- **保護者情報・緊急連絡先**は機密情報として別セクションに表示
  - アンロックボタン押下後、**管理者パスワードまたは編集者パスワード**のいずれかを入力することで30分間閲覧・編集可能
  - 閲覧・編集可能フィールド: 保護者氏名・続柄・携帯番号・緊急連絡先氏名・続柄・電話番号

### 6.4 部員削除

- 部員一覧から「削除」ボタンをクリック
- 確認ダイアログ表示後、`active = 0` に更新（論理削除）

### 6.5 CSVインポート（members_import.php）

**CSVフォーマット**（文字コード: UTF-8 または SJIS 自動判定）:

```
氏名,学年,背番号,所属校,身長,リバビブ番号,青ビブ番号,性別,ローマ字
山田太郎,6,10,犬蔵小学校,145,5,10,男子,TARO
```

- ヘッダー行はスキップ（1行目が数字でなければスキップ）
- 既存部員との重複チェックなし（追加のみ）
- 文字コードはファイル全体を読み込んで自動検出（SJIS-win / EUC-JP / UTF-8）

### 6.6 CSVエクスポート（members_export.php）

- エクスポートする列をチェックボックスで選択（学年・姓・名は必須）
- 選択可能列: 学年・背番号・姓・名・ローマ字・性別・所属校・身長・リバビブ・青ビブ・練習当番・試合当番
- UTF-8 BOM付きCSVで出力（Excelで文字化けしない形式）
- ファイル名: `sugaomax_member_YYYYMMDDhhmm.csv`

## 9. 当番管理機能（duty.php）

### 9.1 概要

- 練習当番（A〜J の10グループ）と試合当番（1〜4 の4グループ）を一覧表示
- `has_sibling = 1` の部員は当番対象外として表示されない
- 各グループに所属する部員数を表示

### 9.2 当番の変更

- 部員カードをドラッグ&ドロップして別グループに移動可能（マウス・タッチ両対応）
- 同一グループ内の並び替えも対応
- 変更はAJAX（`update_duty` アクション）でリアルタイムに保存
- 並び替えはAJAX（`reorder_duty` アクション）で保存
- 編集者以上のロールが必要（閲覧者は変更不可）

### 9.3 未割り当て行

- 各当番表の末尾に「**未**」行を表示し、どのグループにも属していない部員をまとめて表示
- 試合当番の未割り当て行には **4年生以上** の部員のみ表示（3年生以下は試合当番の対象外）
- 未割り当て行もドラッグ&ドロップのドロップ先として機能し、グループから外す操作が可能

### 9.4 同姓部員の表示

- 当番表内で**同じ姓を持つ**部員が複数いる場合（`has_sibling = 0` の部員間で判定）、氏名を「**姓（名）**」形式で表示する
- 同姓でも `has_sibling = 1` の部員は除外して判定するため、兄弟在籍による同姓は表示に影響しない

### 9.5 Excelエクスポート

- 「CSVダウンロード」ボタンから練習当番・試合当番それぞれをExcel（`.xlsx`）ファイルとしてダウンロード可能
- 出力形式: 1グループ1行、メンバー列に `⑥山田 ⑤鈴木` のように学年丸数字＋姓を半角スペース区切りで並べる
- グループ内の並び順は当番表の表示順（`duty_order`）と一致
- メンバー0名のグループも行は出力される（メンバー列は空）
- 未割り当て部員はエクスポートに含まない
- ファイル名: `練習当番_YYYYMMDDhhmm.xlsx` / `試合当番_YYYYMMDDhhmm.xlsx`
- 実装: ZipArchive 不使用、純粋PHPによるZIPバイナリ生成（Office Open XML形式）

## 10. 活動予定機能（schedule.php）

### 10.1 概要

A4横（1122×794px）レイアウトで2か月分の活動カレンダーを表示し、JPG画像として出力できる機能。全ロールがアクセス可能。

### 10.2 レイアウト構成

```
[ヘッダー: ロゴ・タイトル・凡例・対象期間・生成日]
─────────────────────────────────────────────────
[カレンダー（月1）] │ [カレンダー（月2）]
─────────────────────────────────────────────────
[注意事項テキスト] │ [練習当番表] │ [試合当番表]
```

### 10.3 カレンダー表示仕様

- 月曜始まりの週グリッド（月〜金: 等幅、土日: 1.5倍幅）
- イベントはカテゴリー色の背景でセル内に表示
- 時刻あり: イベント名 + 開始〜終了時刻を表示（例: `9:00〜11:00`）
- イベント背景色はDB設定値を白方向に45%混合して表示（視認性確保）
- 月の切替はUI上のセレクトボックスで操作（デフォルト: 当月・翌月）
- 「↺ 更新」ボタンでキャッシュをバイパスして最新データを強制取得

### 10.4 iCalデータ取得・キャッシュ（api/ical_fetch.php）

- `webcal_sources` テーブルに登録されたURLからサーバーサイドでiCalデータを取得
- TLS 証明書検証を有効化（`CURLOPT_SSL_VERIFYPEER => true` / `CURLOPT_SSL_VERIFYHOST => 2`）
- RFC 5545 仕様に準拠したパース（行折り返し展開、VEVENT抽出）
- UTC日時はJST（+9h）に変換
- 終日イベント（DATE型）の複数日展開に対応
- 取得結果をJSONで返す: `{events: [{date, summary, time_start, time_end, category, bg, text}]}`

**サーバーサイドキャッシュ:**

| 項目 | 内容 |
| ---- | ---- |
| 保存先 | `DATA_DIR/cache/ical/{md5}.json` |
| キャッシュキー | 全ソースURL + month1 + month2 の MD5 |
| TTL | 300秒（5分） |
| 強制再取得 | `?force=1` パラメータでキャッシュ削除後に再取得 |
| stale fallback | 全ソース取得失敗時に直近キャッシュが残っていればそれを返す |

### 10.5 webcalソース管理（settings.php）

- カテゴリー名・URL・背景色・文字色を設定して登録
- 登録済みソースの削除・表示順の変更（↑↓ボタン）
- `webcal://` URLは自動的に `https://` に変換して取得

### 10.6 注意事項テキスト（settings.php）

- `contenteditable` ベースの簡易WYSIWYGエディタ
- 対応書式: 太字（`execCommand('bold')`）、文字色（`execCommand('foreColor')`）
- 保存時に `strip_tags()` でサニタイズ（許可タグ: `<b><strong><span><br><div><p><font>`）
- 保存値は `app_settings` テーブルの `schedule_text` キーに格納
- プレーンテキストの既存データは改行を `<br>` に変換して後方互換を維持

### 10.7 JPG出力

- html2canvas 1.4.1 でキャプチャ（scale: 2 で高解像度）
- ダウンロードファイル名: `活動予定_YYYY-MM.jpg`
- 同名ファイルを再生成した場合、古いダウンロードリンクを置き換え（`URL.revokeObjectURL` でメモリ解放）

### 10.8 外部ライブラリ

| ライブラリ | バージョン | 読み込み優先順 |
| ---------- | ---------- | -------------- |
| html2canvas | 1.4.1 | `js/html2canvas.min.js` が存在すればローカル優先、なければ SRI 付き CDN |
| jsPDF | 2.5.1 | `js/jspdf.umd.min.js` が存在すればローカル優先、なければ SRI 付き CDN |

ローカルに配置する場合は cdnjs から対応バージョンをダウンロードして `js/` 配下に置くだけで自動切替される。

## 12. セキュリティ

- SQLインジェクション対策: 全SQLにプリペアドステートメントを使用
- XSS対策: 全出力に `h()` 関数（`htmlspecialchars`）を適用
- CSRF対策: フォーム送信・AJAX処理に `verify_csrf()` を適用
- 認証: 全ページの先頭で `require_login()` を呼び出し
- 権限制御: 操作レベルに応じて `require_editor()` / `require_admin()` を使用
- 機密情報保護: 保護者・緊急連絡先情報は追加パスワード確認後に閲覧可能（30分有効）
- セッション固定化対策: ログイン・認証時に `session_regenerate_id(true)` を実行
- TLS検証: webcal取得時の `CURLOPT_SSL_VERIFYPEER => true` / `CURLOPT_SSL_VERIFYHOST => 2` を明示
- 外部JS完全性検証: CDN利用時に SRI（`integrity`）+ `crossorigin="anonymous"` を付与
- 検索エンジン拒否: 全ページで `X-Robots-Tag: noindex` ヘッダーを送信
