# 菅生マックス チーム管理システム 仕様書

Version 2.0 / 更新日: 2026-04-04

---

## 1. システム概要

菅生マックス（ミニバスケットボールチーム）の部員情報を一元管理し、試合ごとの公式メンバー表をブラウザ上で作成・印刷できるWebアプリケーションです。

### 目的

- Excelや手書きによるメンバー表作成作業を廃止し、ブラウザから誰でも簡単に作成できるようにする
- 部員情報（氏名・学年・背番号・性別・所属校・身長・ビブス番号等）を一元管理する
- 複数のコーチ・保護者が同じデータにアクセスできる共有管理を実現する
- 試合記録を蓄積し、過去のメンバー表をいつでも参照できる
- 公式バスケットボールスコアシート様式で印刷できる

### 技術スタック

| 項目           | 内容                                           |
| -------------- | ---------------------------------------------- |
| 言語           | PHP 8.2                                        |
| データベース   | SQLite 3（ファイルベース、別途DBサーバー不要） |
| フロントエンド | HTML5 / CSS3 / Vanilla JavaScript              |
| 動作環境       | Apache on Linux (Docker)                       |
| 認証方式       | セッションベース パスワード認証                |
| 対応ブラウザ   | Chrome 推奨（印刷機能はChrome使用を前提）      |

## 4. データベース仕様

データベースはSQLite3を使用し、`data/minibasket.db` に保存されます。テーブルおよびカラムは初回アクセス時・カラム未存在時に自動生成・マイグレーションされます。

### 4.1 members テーブル（部員）

| カラム名          | 型      | 必須 | 説明                                       |
| ----------------- | ------- | ---- | ------------------------------------------ |
| `id`              | INTEGER | PK   | 自動採番                                   |
| `name`            | TEXT    | ○    | 部員氏名                                   |
| `grade`           | INTEGER | ○    | 学年（1〜6）                               |
| `number`          | INTEGER |      | 背番号（0〜99）                            |
| `school`          | TEXT    |      | 所属小学校名                               |
| `height`          | INTEGER |      | 身長（cm, 80〜220）                        |
| `active`          | INTEGER |      | 在籍フラグ（1: 在籍 / 0: 論理削除）        |
| `created_at`      | TEXT    |      | 登録日時（自動セット）                     |
| `reversible_bibs` | INTEGER |      | リバーシブルビブス番号（0〜99、未設定は0） |
| `blue_bibs`       | INTEGER |      | 青ビブス番号（0〜99、未設定は0）           |
| `license_no`      | TEXT    |      | ライセンスNo.（任意）                      |
| `gender`          | TEXT    |      | 性別（`男子` / `女子`）                    |
| `romaji`          | TEXT    |      | 名前のローマ字表記（名前のみ、例: TARO）   |

> `active = 0` で論理削除。物理削除は行わないため、データは保持されます。

### 4.2 matches テーブル（試合）

| カラム名          | 型      | 必須 | 説明                                      |
| ----------------- | ------- | ---- | ----------------------------------------- |
| `id`              | INTEGER | PK   | 自動採番                                  |
| `match_date`      | TEXT    | ○    | 試合日（YYYY-MM-DD）                      |
| `opponent`        | TEXT    | ○    | 対戦相手チーム名                          |
| `venue`           | TEXT    |      | 会場名                                    |
| `match_type`      | TEXT    |      | 種別（練習試合 / 公式戦 / 大会 / その他） |
| `note`            | TEXT    |      | 備考（集合時間・持ち物等）                |
| `created_at`      | TEXT    |      | 登録日時（自動セット）                    |
| `coach`           | TEXT    |      | コーチ名                                  |
| `assistant_coach` | TEXT    |      | アシスタントコーチ名                      |

### 4.3 match_members テーブル（試合メンバー）

| カラム名     | 型      | 必須 | 説明                                |
| ------------ | ------- | ---- | ----------------------------------- |
| `id`         | INTEGER | PK   | 自動採番                            |
| `match_id`   | INTEGER | ○    | 試合ID（matches.id 参照）           |
| `member_id`  | INTEGER | ○    | 部員ID（members.id 参照）           |
| `position`   | TEXT    |      | ポジション（PG / SG / SF / PF / C） |
| `sort_order` | INTEGER |      | 表示順（ドラッグ&ドロップで設定）   |

> 試合メンバーを保存するたびに該当試合の全レコードを削除・再挿入します（上書き保存方式）。

## 5. 認証仕様

- チーム共通の単一パスワードによる認証
- ログイン成功後、`$_SESSION['logged_in'] = true` をセット
- 全ページの先頭で `require_login()` を呼び出し、未認証の場合は `login.php` へリダイレクト

### パスワードの変更方法

`includes/db.php` の冒頭にある定数を変更します。

```php
define('APP_PASSWORD', '新しいパスワード');
```

## 6. 部員管理機能

### 6.1 部員一覧（members.php）

- **性別タブ**: ALL / 男子 / 女子（各タブに件数表示）
- **所属校タブ**: ALL / 犬蔵小学校 / 菅生小学校 / 稗原小学校 / その他（各タブに件数表示）
- 2つのタブは**組み合わせフィルター**対応（例: 男子 + 犬蔵小学校）
- **ソート機能**: 学年・背番号・氏名・性別・所属校・リバビブ・青ビブ（昇順/降順切替）
- デフォルト並び順: 学年降順（6年から）
- 氏名クリックで部員編集ページへ遷移
- 表示列: 学年・背番号・氏名・性別・所属校・リバーシブルビブス・青ビブス

### 6.2 部員追加（member_add.php）

入力フィールド:

| フィールド             | 必須 | 備考                 |
| ---------------------- | ---- | -------------------- |
| 氏名                   | ○    |                      |
| ローマ字表記           |      | 名前のみ（例: TARO） |
| 学年                   | ○    | 1〜6年               |
| 性別                   |      | 男子 / 女子          |
| 背番号                 |      | 0〜99                |
| 所属小学校             |      |                      |
| 身長（cm）             |      | 80〜220              |
| リバーシブルビブス番号 |      | 0〜99                |
| 青ビブス番号           |      | 0〜99                |

### 6.3 部員編集（member_edit.php）

- 部員追加と同じフィールドを編集可能
- ページタイトルに部員名を表示
- 更新後も同ページに留まりメッセージを表示

### 6.4 部員削除

- 部員一覧から「削除」ボタンをクリック
- 確認ダイアログ表示後、`active = 0` に更新（論理削除）

### 6.5 CSVインポート（members_import.php）

**CSVフォーマット**（文字コード: UTF-8 または SJIS 自動判定）:

```
氏名,学年,背番号,所属校,身長,リバビブ番号,青ビブ番号,性別,ローマ字
山田太郎,6,10,犬蔵小学校,145,5,10,男子,TARO
```

- ヘッダー行はスキップ（1行目が数字でなければスキップ）
- 既存部員との重複チェックなし（追加のみ）
- 文字コードはファイル全体を読み込んで自動検出（SJIS-win / EUC-JP / UTF-8）

## 8. メンバー表機能（match_sheet.php）

### 8.1 作成フロー

1. 試合一覧から試合を選択し「メンバー表」ボタンをクリック
2. 画面左側の選手選択パネルで参加選手にチェックを入れる
3. 各選手のポジション（PG / SG / SF / PF / C）を選択（任意）
4. 選手をドラッグ&ドロップで任意の順番に並び替え
5. 「保存する」ボタンでデータベースに保存
6. 「印刷」ボタンで印刷ダイアログを起動

### 8.2 選手選択パネル

- 性別タブ（ALL / 男子 / 女子）でフィルタリング可能
- 選手は学年降順（6年から）で表示
- チェック状態はタブ切替後も保持

### 8.3 プレビュー表示

右側プレビューに選択中の選手一覧を表示:

| 項目 | 内容                      |
| ---- | ------------------------- |
| ☰   | ドラッグ&ドロップハンドル |
| #    | 背番号                    |
| 氏名 | 選手の氏名                |

### 8.4 印刷レイアウト（公式バスケットボールスコアシート様式）

- **用紙**: A4縦
- **配置**: 1枚の用紙に同じメンバー表を **2列×2行（計4枚）** 印刷
- **余白**: 3mm（Chrome印刷設定: 余白なし・拡大縮小100%推奨）

#### 各シートの構造（12列）

| 列          | 内容                | 幅      |
| ----------- | ------------------- | ------- |
| 1           | №（行番号）         | 5.5mm   |
| 2（名前列） | 選手氏名            | 38.5mm  |
| 3           | No.（背番号）       | 5.5mm   |
| 4〜7        | 出場時限 ①②③④       | 各5.5mm |
| 8〜12       | ファウル １２３４５ | 各5.5mm |

#### ヘッダー部（行1〜3）

| 列                       | 内容                   |
| ------------------------ | ---------------------- |
| 列1（1列）               | チーム：／Team         |
| 列2〜4（3列, rowspan=3） | 菅生マックス           |
| 列5〜7（3列, rowspan=3） | （　　　）             |
| 列8〜12（5列）           | タイムアウト ／ ①②③④OT |

#### 列ヘッダー部（行4〜5）

- №・選手氏名(Players)・No.・出場時限(①②③④)・ファウル(１２３４５)

#### 選手行（行6〜25）

- 20行固定（選手未満の場合は空行）

#### コーチ行（行26〜27）

- コーチ：（名前）/ A. コーチ：（名前）
- 右側5列（ファウル列）は個別セルの罫線を維持

## 9. セキュリティ

- SQLインジェクション対策: 全SQLにプリペアドステートメントを使用
- XSS対策: 全出力に `h()` 関数（`htmlspecialchars`）を適用
- 認証: 全ページの先頭で `require_login()` を呼び出し

## 10. セットアップ手順

### 動作要件

- PHP 8.2以上（pdo_sqlite 拡張が有効であること）
- Apache Webサーバー
- `data/` ディレクトリへの書き込み権限

### パスワード変更

`includes/db.php`:

```php
define('APP_PASSWORD', 'sugaomax2024'); // ← ここを変更
```

### 印刷時の推奨設定（Chrome）

- 余白: なし（または最小）
- 拡大縮小: 100%（デフォルト）
- 用紙サイズ: A4縦
